Modelos Arquitectónicos

MVC

Modelo-Vista-Controlador: separa la aplicación en Modelo, Vista y Controlador para mayor mantenibilidad y testeo.

MVP

Modelo-Vista-Presentador: mejora la separación entre la UI y la lógica de presentación, facilitando pruebas unitarias.

MVVM

Modelo-Vista-ViewModel: introduce un ViewModel que actúa como intermediario entre la UI y la lógica de negocio.

Clean Architecture

Organiza el software en capas independientes, permitiendo cambios en la UI o la base de datos sin afectar la lógica de negocio.
